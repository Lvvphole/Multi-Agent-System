# How to get this into the repo (you open the PR; the assistant cannot)

Pushing to a remote is an external/irreversible action, gated by the contract
(autonomy 2 external / HUMAN_IN_THE_LOOP). The assistant produces files; you
commit and open the PR.

## What gets committed (source + design deliverables)
- CONTRACT.md
- harness/ (all .py)
- run_stage0.py ... run_stage4.py
- tests/ (test_harness.py, test_results.log)
- _authored/ (per-stage Executor drafts)
- artifacts/ (gated design deliverables: schemas, models, matrices, registers)
- decisions/ (DR-001..DR-003)
- project_memory.md, README.md, .gitignore, COMMIT_GUIDE.md

## What is git-ignored (mutable runtime state, regenerated by running)
- state_store.json, artifact_manifest.json, idempotency.json
- gate_state_log.jsonl, judge_session_log.jsonl, human_approval_log.jsonl

## Suggested commands (run locally after downloading the files)
git clone https://github.com/Lvvphole/Multi-Agent-System.git
# copy these files into the clone, then:
git checkout -b feat/contract-harness-stages-0-4
git add CONTRACT.md harness run_stage*.py tests _authored artifacts decisions \
        project_memory.md README.md .gitignore COMMIT_GUIDE.md
git commit -m "Contract v2.1.0 + deterministic harness; stages 0-4 gated (16 tests)"
git push -u origin feat/contract-harness-stages-0-4
# then open the PR on GitHub

## Note on artifacts/ vs runtime
artifacts/ holds the verified design deliverables (keep). The manifest/logs that
*describe* them are runtime lineage (ignored). At Stages 12-13 the harness emits
verification_report.md / evaluation_report.md / release_gate_report.md — those
are the durable, committable evidence snapshots.
